## 1. Onboarding & Authentication

**Screens:** 4 | **Purpose:** First-run experience, profile setup, and access control

### User flow

```
Splash / brand → SSO / email login → Registration verify → Profile builder → Interest tagging → Home
```

---

### 1.1 Splash & SSO screen

| Feature | Detail |
|---|---|
| Google / LinkedIn / Apple SSO | OAuth 2.0 single sign-on; LinkedIn preferred given audience |
| Magic link email login | Passwordless; link expires in 15 minutes |
| Biometric unlock on return | Face ID / fingerprint for repeat sessions |
| Registration badge validation | Cross-checks email against attendee registration database |

**States:**
- `Unregistered` → prompt to register on event website
- `Registered, no account` → create account flow
- `Existing user` → biometric or SSO straight to Home

---

### 1.2 Profile builder

| Feature | Detail |
|---|---|
| Executive headshot upload | Camera capture or gallery; auto-cropped to 1:1 |
| Company, title, sector | Free text + sector taxonomy (dropdown) |
| Interest & deal-size tagging | Multi-select tags: invest / partner / hire / learn; deal range selector |
| Profile visibility controls | Public / connections only / hidden; granular per field |
| LinkedIn profile sync (optional) | One-tap import of headline, photo, and experience |

**Validation rules:**
- Headshot required for networking features to unlock
- Minimum 3 interest tags required to activate AI matching
- Title and company are mandatory

---

### 1.3 Digital access pass

| Feature | Detail |
|---|---|
| QR digital access badge | Dynamic QR code tied to registration tier; required for physical venue entry |
| NFC tap-to-share contact | Tap phones to exchange contact cards |
| Tier-gated content flags | VIP / standard / speaker badges render visually distinct; gates certain sessions |
| Session capacity alerts | Push alert when a session the user bookmarked opens capacity |

> **Note:** The digital pass must function offline (cached QR) in case of poor venue connectivity.

---

### 1.4 Interest tagging screen

- Attendee selects primary objectives (up to 5):
  - Raise capital / deploy capital
  - Find strategic partners
  - Recruit talent
  - Learn about a topic area (freeform text)
  - Explore new markets
- Selections feed directly into the AI matchmaking engine
- Can be updated at any time from Settings > Profile

## 2. Home & Personalised Feed

**Widgets:** 5 | **Purpose:** Command centre for the day's experience

---

### 2.1 Now & next strip

| Feature | Detail |
|---|---|

Refreshes each morning at 07:00 local time and after major schedule changes.

|---|---|

---

### 2.3 Activity feed

| Feature | Detail |
|---|---|

---

### 2.4 My stats widget

Visible on Home and on the Profile screen.

| Metric | Description |
|---|---|

---

### 2.5 Quick-action bar

Persistent bottom navigation with four icons:

1. **Home** — returns to the feed

## 3. Agenda & Sessions

**Screens:** 6 | **Purpose:** Full programme, personalisation, live translation, post-session content

### User flow

```
Browse agenda → Filter / search → Session detail → Add to schedule → My schedule view → Live session → Post-session
```

---

### 3.1 Programme browser

| Feature | Detail |
|---|---|
| Filter by track, format, day, speaker | Multi-select filters; filters persist across app restarts |
| Full-text search | Indexed across session titles, descriptions, speakers, and companies |
| AI "recommended for you" tag | Green badge on sessions matching the user's profile and intent |
| Bookmark to shortlist | Heart icon; shortlisted sessions aggregated on My Schedule |
| Attendee count & capacity indicator | Live count; colour-coded: green (plenty of space) / amber (filling up) / red (waitlist only) |

---

### 3.2 Session detail page

| Feature | Detail |
|---|---|
| Speaker bios | Photo, title, company; tapping opens attendee profile with connect button |
| Add to personal schedule | Conflict warning if another session overlaps |
| Room + one-tap navigation | Deep-links into the indoor navigation module |
| Pre-read materials download | PDF / slides available before the session |
| Share session with a contact | Sends a link via in-app message or external share sheet |

---

### 3.3 My schedule view

- Timeline view (default) and list view toggle
- Colour-coded by track
- Drag to reorder uncommitted bookmarks
- Export to external calendar (Google / Apple / Outlook)
- Conflict detection highlights clashes in red

---

### 3.4 Live session screen *(must-have)*

| Feature | Detail |
|---|---|
| Live in-ear audio translation | 8 languages minimum: Arabic, Mandarin, French, German, Spanish, Portuguese, Japanese, English |
| Real-time captions overlay | AI-generated; displayed as subtitle bar at bottom of screen |
| In-app note-taking | Per-session; notes exportable as PDF or pushed to post-session summary |
| Q&A submission & upvote | Attendees submit text questions; moderator curates; popular questions rise |
| Live audience polls | Displayed by speaker/moderator; results shown in real time |
| Check-in | Manual tap or automatic via indoor location dwell-time trigger |

**Translation technical note:** Requires integration with a real-time audio pipeline (e.g. Speechmatics, Deepgram, or Azure Cognitive Services). This is the most technically complex feature in the app.

---

### 3.5 Post-session screen

| Feature | Detail |
|---|---|
| AI-generated session summary | 200–400 word synthesis of key points; generated within 5 minutes of session end |
| Slides & materials delivery | Session deck pushed in-app; downloadable as PDF |
| On-demand replay | Available if session is recorded; embeds in-app video player |
| Rate session & speaker | 1–5 stars; optional free-text comment; anonymous by default |

---

### 3.6 Conflict & capacity handling states

| State | Behaviour |
|---|---|
| Schedule conflict warning | Modal alert on "Add to schedule"; offers to remove conflicting session or add to waitlist |
| Waitlist join | One-tap join; position displayed; push notification on capacity release |
| Capacity released alert | Push + in-app banner; 10-minute window to confirm attendance |

## 4. AI Networking & Matchmaking

> **Must-have feature**

**Screens:** 7 | **Purpose:** Intent-based attendee discovery and connection

### Matching flow

```
Intent survey → AI generates ranked matches → Review profile card → Send connection request → In-app messaging → Schedule meeting
```

---

### 4.1 Match engine

| Feature | Detail |
|---|---|
| Intent declaration | Choose primary intent: invest / partner / hire / learn |
| Sector, deal stage, company size filters | Narrow the match pool before AI ranks results |
| AI match score with explanation | Score out of 100; one-sentence rationale ("Both focused on Series B fintech in MENA") |
| Daily refresh | Matches recalculate at 07:00 and 13:00 each event day |
| "Who should I meet about X?" query | Natural language input → filtered match list (routes to AI Assistant) |

**Matching signals used:**
- Profile intent tags
- Session bookmarks and attendance
- Search queries
- Mutual connections
- Company and sector overlap
- Deal size compatibility

---

### 4.2 Attendee profile card

| Feature | Detail |
|---|---|
| Full executive profile | Photo, name, title, company, bio, sector |
| Mutual connections | Shows shared connections with this event's attendee list |
| Sessions they're attending | Visible if the attendee's schedule is set to public |
| Send message | Opens encrypted 1:1 chat thread |
| Request meeting | Opens the meeting scheduler with this attendee pre-filled |
| Save to watchlist | Private list; no notification sent to the other party |
| Block / report | Available via overflow menu; reported profiles reviewed by organiser |

---

### 4.3 Connections list

- Tabs: Matches / Requests sent / Requests received / Connected / Watchlist
- Search within connections by name or company
- Export all connections as CSV or push to CRM

---

### 4.4 Messaging

| Feature | Detail |
|---|---|
| End-to-end encrypted 1:1 messaging | TLS in transit; messages stored encrypted at rest |
| AI suggested icebreaker | One-tap generated opening message based on shared context |
| Inline meeting invite from chat | "Schedule meeting" button inside thread |
| Share session & documents in chat | Deep-link to session or attach PDF |
| Do-not-disturb mode | Silences message notifications; status visible to senders |
| Read receipts | Optional; can be disabled in Settings |

---

### 4.5 In-person connect

| Feature | Detail |
|---|---|
| Scan badge to connect | Camera opens to QR scanner; instant connection request sent |
| NFC tap alternative | Tap phones to exchange; works without camera permission |
| Auto-save digital business card | Name, title, company, email, phone saved to in-app contacts |
| Export to CRM | One-tap push to Salesforce, HubSpot, or generic CSV |

---

### 4.6 Attendee directory

- Browse all attendees (if organisers enable this)
- Filter by sector, country, company size, badge tier
- Search by name or company
- Visibility is always subject to the individual attendee's profile privacy settings

---

### 4.7 Post-event networking window

- Connection features remain active for 6 months post-event
- Matches refresh weekly during the post-event window
- In-app messaging available throughout the post-event period

## 5. Meeting Scheduler

> **Must-have feature**

**Screens:** 5 | **Purpose:** Pre-event and on-site 1:1 meeting management

### Booking flow

```
View availability → Propose time slots → Accept / counter-propose → Confirmed + location assigned → 15-min reminder → Check-in → Post-meeting notes
```

---

### 5.1 Availability management

| Feature | Detail |
|---|---|
| Block out personal unavailable times | Manual time blocking; e.g. "Do not book 12:00–13:30" |
| Two-way calendar sync | Google Calendar and Outlook; bi-directional; updates in real time |
| Slot duration options | 30 / 45 / 60 minutes |
| Meeting blackout windows | Organisers can define no-meeting zones (e.g. during keynotes) |
| Open for meetings toggle | Global switch to pause incoming requests without explanation |

---

### 5.2 Meeting rooms browser

| Feature | Detail |
|---|---|
| VIP lounge & breakout room directory | Photos, capacity, AV availability, catering options per room |
| Real-time room availability | Greyed out when booked; live sync with booking system |
| Navigate to room | One-tap from meeting confirmation opens indoor navigation |
| Arrival check-in | Check in when you arrive; notifies the other party |

---

### 5.3 My meetings list

| Feature | Detail |
|---|---|
| Chronological meeting list | Today / upcoming / past tabs |
| Pre-meeting agenda notes | Private notepad per meeting; visible only to you |
| Cancel with auto-notification | Cancelling sends notification to other party; prompts for a reason |
| Post-meeting AI action items | After the meeting ends, AI prompts for notes and generates action items |
| Push summary to CRM | Send meeting notes and action items directly to Salesforce / HubSpot |

---

### 5.4 Meeting states

| State | Visual indicator | Description |
|---|---|---|
| Confirmed | Green | Both parties accepted; room assigned |
| Pending response | Amber | Awaiting the other party's acceptance |
| Rescheduled | Blue | New time proposed; awaiting re-confirmation |
| Declined | Red | Other party declined |
| Completed | Grey | Meeting time has passed; post-meeting prompt shown |
| Cancelled | Strikethrough | Cancelled by either party |

---

### 5.5 Pre-event meeting scheduling

- Available from 4 weeks before the event
- Attendees can browse the directory and request meetings before arriving
- Time slots anchored to event days and venue hours
- Organisers can run a curated matchmaking round: submit mutual interest → confirmed meetings placed in both calendars

## 6. Indoor Navigation & Venue

> **Must-have feature**

**Screens:** 4 | **Purpose:** Venue mapping, turn-by-turn wayfinding, and points of interest

---

### 6.1 Venue map

| Feature | Detail |
|---|---|
| Multi-floor interactive map | Pinch-to-zoom; floor selector on side rail |
| Blue-dot live location | Bluetooth Low Energy (BLE) beacon triangulation; Wi-Fi fallback |
| Filter POIs | Toggle: session rooms / networking lounges / exits / catering / prayer rooms / medical |

| Accessibility-friendly routes | Avoids stairs; flags step-free paths |
| ETA to next session | Live walking time estimate to booked room |

### 6.4 Transport & external logistics

## 7. AI Summit Assistant

> **Must-have feature**

**Availability:** Persistent floating button; accessible from any screen

**Purpose:** Conversational executive concierge that knows the attendee's full context

---

### 7.1 Chat interface

| Feature | Detail |
|---|---|
| Voice input | Speech-to-text; tap microphone to speak |
| Text input | Standard keyboard; supports multiple languages |
| Multilingual responses | Responds in the language the user writes in |
| Suggested prompts | Contextual quick-reply chips updated throughout the day |

**Example queries the assistant handles:**

- "What should I do in the next 2 hours?"
- "Who at this event is raising a Series B in energy?"
- "How do I get to the Meridian Lounge?"
- "Summarise the keynote I missed this morning"
- "Draft a message to [name] asking for 20 minutes"
- "Show me my meetings for tomorrow"

---

### 7.2 Smart outputs

| Output | Description |
|---|---|
| Session summary | 200–400 word synthesis of any session the user attended or missed |
| ROI justification letter | Personalised letter for the user to share with their team or board making the case for event value |
| Post-event action item digest | End-of-day or end-of-event digest of all action items from notes and meetings |
| Key themes & signals | Daily debrief: top 3 themes the user encountered across their schedule |
| Icebreaker message draft | Tailored opening message for any attendee in the directory |

---

### 7.3 Context & memory

| Feature | Detail |
|---|---|
| Full attendee context | Knows the user's schedule, confirmed meetings, bookmarks, and stated interests |
| Privacy scope | Data used only within the app session; not used for model training |
| Contextual adaptation | Suggestions evolve as the day progresses and events are attended |
| Escalation to organiser | If the assistant cannot answer, it routes to the organiser support chat |

## 9. Cross-Cutting Notes

### Architecture dependencies

| Dependency | Modules affected | Notes |
|---|---|---|
| BLE beacon infrastructure | 6 (Navigation), 5 (Meeting rooms), 2 (Now & next) | Must be provisioned before the event; minimum one beacon per 15m |
| Real-time audio pipeline | 3.4 (Live translation) | Speechmatics / Deepgram / Azure Cognitive Services; highest technical complexity |
| Calendar API (Google + Outlook) | 5.1, 3.3 | OAuth scopes for read + write access |
| CRM API (Salesforce / HubSpot) | 4.5, 5.3 | OAuth; scopes for contact create and activity log |
| AI / LLM backend | 2.2, 4.1, 5.3, 7 | Must support function calling to query live event data |

---

### Feature interdependencies

The four must-have features are tightly coupled and should be built in a single integrated sprint:

- **AI matchmaking** is only useful if **meeting scheduler** exposes real-time availability
- **Indoor navigation** becomes contextual (not just a map) when triggered from session detail pages and meeting confirmations
- **AI assistant** needs access to schedule, meeting, and location context to answer useful queries
- **Live translation** requires a network-quality guarantee; fallback (caption-only mode) must be implemented

### Platform requirements

- **iOS:** iOS 16 and above
- **Android:** Android 10 (API 29) and above
- **Offline mode:** Agenda, venue map, digital pass, and saved contacts must function without connectivity

- **Data residency:** Confirm with legal team for events in the EU (GDPR) and Saudi Arabia (PDPL)

Wood Mackenzie's metals & mining platform is essentially a premium research, data, and analytics intelligence product for the mining industry — covering price forecasts, supply/demand, cost benchmarking, asset data, and market outlooks across copper, lithium, nickel, cobalt, gold, coal, and more. Here's how you can build something **better** using modern AI and agentic capabilities:

---

## What Wood Mackenzie Does (and Where It Falls Short)

Its core offering is: curated analyst reports, a data platform (Lens), asset-level cost benchmarks, and commodity price forecasts — all behind a high-cost subscription wall, updated on analyst cycles, and largely static (you read reports, you don't interact with the data).

**The gaps AI can exploit:**

- Data is expensive, siloed, and not real-time
- No conversational interface — users read PDFs and static dashboards
- No personalized alerting or reasoning
- No integration with a user's own internal data
- Reports are written by analysts on a schedule, not on demand

---

## How to Build a Better Product with AI

### 1. Agentic Data Aggregation (Replace Expensive Proprietary Data)

Instead of paying hundreds of analysts, deploy **AI agents** that continuously scrape, parse, and synthesize:

- **Public mine disclosures**: EDGAR, SEDAR, ASX, JSE filings (production reports, cost data)
- **Government sources**: USGS, World Bank commodity data, national mining ministries
- **Exchange data**: LME, CME, SHFE real-time and historical prices via APIs
- **Satellite + geospatial**: Planet Labs or Sentinel-2 imagery for monitoring mine activity (stockpiles, operational status)
- **Port/shipping data**: AIS vessel tracking APIs for trade flow intelligence
- **News + social signals**: Financial news, regulatory announcements, labor disputes

Use a multi-agent pipeline (e.g., with LangGraph or Claude's tool-use) where specialized agents handle each data source and a synthesis agent assembles insights.

### 2. Conversational Intelligence Layer

Wood Mackenzie makes you navigate menus to find a report. Build a **natural language interface** over your data:

> *"What's the 5-year copper supply outlook given current Chilean water stress and Chinese smelter capacity?"*

This is now highly buildable with RAG (Retrieval-Augmented Generation) over your structured data + document corpus, connected to an LLM like Claude. Users get analyst-quality answers on demand, not on a report schedule.

### 3. Personalized Agentic Alerts

Build agents that monitor for conditions a specific user cares about and proactively surface insights:

- "Alert me when lithium carbonate spot price diverges more than 15% from WM's forecast"
- "Monitor Pilbara Minerals quarterly reports and flag any changes to C1 cost guidance"
- "Watch for ESG incidents at any copper mine in my portfolio"

This is currently absent from Wood Mackenzie's offering entirely.

### 4. Asset-Level AI Cost Modeling

WoodMac's cost curves are a core product. You can replicate and improve them by:

- Ingesting public AISC/C1 cost data from company filings
- Using AI to extract cost components from annual reports (energy, labor, reagents)
- Building open **cost curve models** users can stress-test with their own assumptions
- Integrating commodity price APIs so the model recalculates in real time

### 5. AI-Generated Reports On Demand

Instead of static PDFs published quarterly, offer **on-demand AI research generation**:

- User selects commodity + geography + timeframe + thesis
- Agentic workflow gathers latest data, runs through models, synthesizes a structured report
- Report is grounded in citations, not hallucinated

Tools: Claude with tool use + web search + a knowledge graph of mining assets.

### 6. Integrate with Users' Internal Data

This is a massive differentiation opportunity. Mining companies and funds have their own models, spreadsheets, and internal databases. Build a platform where they can:

- Upload their own mine models or portfolio data
- Ask questions that blend Wood Mackenzie-equivalent public data **with their private data**
- Get benchmarking ("how does my mine's AISC compare to the industry cost curve?")

This is where agentic AI with MCP-style tool use becomes extremely powerful.

## Tech Stack to Build This

| Layer | What to Use |
|---|---|
| **Data ingestion agents** | Python + Playwright/BeautifulSoup + scheduled Claude agents |
| **Structured data store** | PostgreSQL + TimescaleDB for time-series price data |
| **Document/report store** | Vector DB (Pinecone, Weaviate) for semantic search over filings |
| **AI backbone** | Claude API (tool use, agentic workflows) |
| **Orchestration** | LangGraph or custom multi-agent framework |
| **Frontend** | React dashboard with conversational UI |
| **Alerting** | Background agents + email/Slack/webhook triggers |
| **Price feeds** | Quandl, LME API, Metals Daily |

## Where to Focus for Competitive Advantage

The single biggest lever is **real-time + interactive**. WoodMac is essentially a very expensive, slow, static research service. If you build something that:

1. Updates continuously from public sources
2. Lets users *talk* to the data
3. Integrates with their own workflows
4. Costs 10x less

…you don't need to beat their analyst depth on day one. You win on speed, accessibility, and integration.

A focused vertical — say, **just lithium and battery metals** (huge demand given EV transition) — would be a viable starting wedge with far less capital than trying to match WoodMac's full coverage from day one.

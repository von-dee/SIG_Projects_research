Product Blueprint
Strata — AI-native metals & mining intelligence
A full-stack product specification: architecture, personas, use cases, and business model for a next-generation alternative to Wood Mackenzie.
The core proposition
Wood Mackenzie is a $2B+ intelligence business built on expensive analyst labour and slow publication cycles. Strata replaces the model with continuous agentic data collection, real-time AI synthesis, and a conversational interface — at a fraction of the cost.

What Strata is
An AI-native research and analytics platform that aggregates public mining data, synthesizes it continuously with LLM agents, and lets users query it conversationally — with their own private data blended in.
What Wood Mackenzie is not
Interactive, real-time, or integrable. It sells analyst PDFs and static dashboards updated on human schedules. Zero conversational interface. No private data upload. High switching costs by design.
Strategic wedge
Start narrow, expand fast. Battery metals are undercovered relative to their market growth and have the most urgent demand for real-time data (EV supply chains, government incentive tracking, exchange pricing).

Full application architecture
Seven-layer stack from raw data ingestion through to user-facing interfaces. Each layer is independently scalable and replaceable.
1
Data ingestion layer — agentic crawlers
Scheduled and event-triggered agents collect from public filings (EDGAR, SEDAR, ASX), government databases (USGS, World Bank, national ministries), exchange APIs (LME, CME, SHFE), vessel tracking, and news feeds. Each agent is specialised per source type.
Python · Playwright · BeautifulSoup · APScheduler · AIS API · Planet Labs satellite API · Kafka (event bus)
2
Data normalisation & enrichment
Raw data is cleaned, normalised, and enriched. LLM-powered extraction pulls cost components from unstructured annual reports. A knowledge graph maps relationships between companies, assets, regions, and commodities. Entity resolution deduplicates across sources.
Claude API (extraction) · dbt · Apache Spark · Neo4j (knowledge graph) · OpenStreetMap (geospatial)
3
Storage layer — multi-model
Time-series price data in TimescaleDB. Structured asset/company data in PostgreSQL. Document corpus (filings, reports) in a vector database for semantic retrieval. Knowledge graph in Neo4j. Geospatial mine and deposit data in PostGIS.
PostgreSQL · TimescaleDB · Pinecone (vectors) · Neo4j · PostGIS · S3 (raw file archive)
4
AI orchestration layer — agents + RAG
The core intelligence layer. A multi-agent system with specialised sub-agents (price forecaster, filing extractor, news synthesiser, alert evaluator, report generator). A supervisor agent routes queries, assembles context from all data stores, and manages tool use. RAG retrieval is hybrid: semantic + keyword + structured SQL.
Claude API (claude-sonnet-4) · LangGraph (agent orchestration) · pgvector + Pinecone (retrieval) · MCP tool servers
5
Quantitative models layer
Parameterised supply/demand models per commodity. Cost curve engine that re-renders in real time as input data updates. Monte Carlo simulation for price scenarios. Models are transparent: users can inspect assumptions and override inputs.
Python · NumPy · SciPy · Pandas · Statsmodels · custom supply/demand model modules
6
API & backend services
REST + GraphQL API serving all data to the frontend and third-party integrations. Background worker pool for agentic alert evaluation, scheduled report generation, and data refresh jobs. Auth, multi-tenancy, and private data isolation at this layer.
FastAPI · GraphQL (Strawberry) · Celery · Redis (queue + cache) · Auth0 · PostgreSQL RLS (row-level security)
7
Frontend — dashboard + conversational UI
React SPA with three primary surfaces: a data dashboard (charts, cost curves, price feeds), a conversational research interface (chat over data with citations), and an alerts/workflow management console. Mobile-responsive. White-label option for enterprise.
React · TypeScript · Recharts / D3 · Tailwind · Vercel · WebSockets (live price feeds)
Infra & DevOps
Cloud
AWS primary. ECS (containerised agents), RDS (Postgres), ElastiCache (Redis), S3. Multi-region from year 2.
Observability
Datadog for infra metrics, Sentry for errors, LangSmith for LLM trace logging and prompt performance monitoring.
Security & compliance
SOC 2 Type II from year 1 (critical for enterprise). Data encrypted at rest + in transit. Private data tenancy isolation enforced at DB layer.
Data sources & ingestion strategy
Every data source is public, API-accessible, or scrapeable. No expensive data licences required at launch. The moat is the collection infrastructure, normalisation quality, and historical depth — not the raw source.
Price & exchange data
LME / CME / SHFE
Official spot and futures prices for base metals. LME has a free delayed API; real-time via Quandl or direct LME data subscription (~$2k/yr).
Fastmarkets / Benchmark
Battery-grade lithium, cobalt, and nickel spot assessments. Most critical price series for Phase 1 coverage.
World Bank commodities
Free long-run price series going back 30+ years. Essential for historical model calibration.
Production & company filings
EDGAR / SEDAR / ASX
Quarterly and annual production reports, MD&A sections, cost guidance. The primary source for AISC and C1 extraction. Fully free and API-accessible.
NI 43-101 / JORC reports
Resource/reserve technical reports. AI extracts ore grade, strip ratio, recovery rates, capital cost estimates from PDF.
Mining ministry databases
Production permit databases from Chile (SERNAGEOMIN), DRC, Australia (Geoscience), Indonesia (ESDM). Country-level production volumes.
Alternative & signal data
Satellite imagery
Sentinel-2 (free, ESA) for mine activity monitoring, stockpile change detection. Planet Labs for higher-res commercial upgrade.
AIS vessel tracking
Trade flow intelligence — shipping routes for ore and refined metal. MarineTraffic API for port-level loading/unloading signals.
News & regulatory feeds
Factiva API, Mining.com, S&P Global Commodity Insights RSS for ESG incidents, labour disputes, permit approvals, force majeures.
AI agent architecture
A supervisor-worker multi-agent system built on LangGraph. Each specialist agent has a defined scope, tool access, and output contract. The supervisor routes incoming queries and assembles responses from multiple agents.
Supervisor agent
Receives user queries. Classifies intent (research / data query / alert / report). Decomposes into subtasks. Fans out to specialist agents. Assembles and cites the final response. Handles clarification turns.
claude-sonnet-4LangGraph orchestration
Retrieval agent (RAG)
Hybrid retrieval across vector store (filings, reports), SQL (structured price/production data), and knowledge graph (asset relationships). Reranks and packages context for the synthesis agent.
Pinecone + pgvectorBM25 + semantic
Filing extraction agent
Specialised in parsing mining company filings. Extracts structured cost data, production volumes, guidance changes, and capex from PDFs. Uses chain-of-thought to handle inconsistent reporting formats.
claude-sonnet-4Continuous background job
Price forecast agent
Interfaces with the quantitative S/D model layer. Runs scenarios when asked, explains model drivers, and flags when price is trading significantly above or below model fair value.
Tool use + Python REPL
Alert evaluation agent
Runs as a background job against all subscribed user alert conditions. When triggered, generates a plain-language explanation of what happened, the data evidence, and suggested next steps.
Celery workerRuns every 15 min
Report generation agent
On-demand or scheduled structured report creation. Follows a configurable template (executive summary, supply analysis, demand outlook, price forecast, risk factors). All claims grounded with citations to source data.
claude-sonnet-4PDF/DOCX export
Private data integration
The most defensible enterprise feature. Users upload their own data — mine financial models, portfolio spreadsheets, internal cost assumptions. It is stored in a fully isolated tenant partition, never used to train models, and blended into AI responses only when relevant.
Supported formats
Excel (.xlsx), CSV, PDF (technical reports, board packs), structured JSON via API.
Isolation model
Row-level security in Postgres. Separate vector namespaces per tenant in Pinecone. Private data never leaves the tenant context.
Blended queries
"How does our Atacama asset's AISC compare to the industry cost curve?" merges private + public data in one AI response.

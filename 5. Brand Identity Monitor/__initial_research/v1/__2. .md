**Radio transcription pipeline — how to actually build it**

Community FM stations in Ghana (like Adom FM, Asempa FM, Radio Gold, or dozens of regional stations in mining areas like Bibiani, Obuasi, Tarkwa) broadcast live and some stream online. Here's the full technical approach:

**Step 1 — Capture the stream**

For stations that have an online stream (Icecast/Shoutcast), you can record directly:

```bash
ffmpeg -i http://stream.radiostation.com/live \
  -acodec pcm_s16le -ar 16000 -ac 1 \
  -f segment -segment_time 3600 \
  output_%Y%m%d_%H%M.wav
```

For stations without online streams, you need a physical SDR (Software Defined Radio) receiver like a RTL-SDR dongle (~$25) at a location near the mine. A Raspberry Pi with an RTL-SDR can capture any FM frequency and upload audio chunks to your cloud pipeline automatically.

**Step 2 — Speech-to-text with Whisper**

OpenAI's Whisper runs locally and handles Twi, Hausa, Dagbani, French, and many other local languages — this is critical. You don't send audio to an external API:

```python
import whisper

model = whisper.load_model("large-v3")  # large for local languages
result = model.transcribe(
    "segment_20240115_0800.wav",
    language=None,        # auto-detect language
    task="transcribe",    # or "translate" to go straight to English
    verbose=True
)

print(result["text"])
print(result["language"])  # detected language
```

The `large-v3` model runs comfortably on a single A100 GPU. For cost-efficiency, run batch jobs nightly — capture all day, transcribe overnight.

**Step 3 — Segment by speaker / topic**

Raw transcripts of a 2-hour talk show are noisy. Use pyannote-audio for speaker diarization (who is speaking when) and then chunk into logical segments:

```python
from pyannote.audio import Pipeline

diarization = Pipeline.from_pretrained("pyannote/speaker-diarization-3.1")
diarization_result = diarization("segment.wav")

for turn, _, speaker in diarization_result.itertracks(yield_label=True):
    print(f"{speaker}: {turn.start:.1f}s – {turn.end:.1f}s")
```

This lets you separate a radio host's questions from a community member's call-in response — very different sentiment signals.

**Step 4 — Translation for multilingual content**

If a broadcast is in Twi or Dagbani, translate before sentiment analysis:

```python
from transformers import pipeline

translator = pipeline("translation", 
    model="Helsinki-NLP/opus-mt-tw-en")  # Twi → English
```

Helsinki-NLP has models for many African languages. For rarer languages you may fall back to Meta's NLLB-200 (`facebook/nllb-200-distilled-600M`) which covers 200 languages including many West African ones.

**The other high-value sources you asked about**

Beyond radio, the most underrated sources for mining community intelligence are:

Parliamentary Hansard records are gold. Any time a mining company, mine name, or affected community is mentioned in parliament, you want that captured. Ghana, Zambia, and South Africa all publish Hansard online and it's freely scrapeable. These debates often surface grievances months before they become protests.

NGO and CSO reports from organizations like Wacam (Ghana), ZCCM-IH community monitors (Zambia), or SARW (Southern Africa Resource Watch) contain structured community perception data that no one is currently aggregating systematically. Many publish PDFs — you extract these with `pdfminer` or `pypdf2` and run them through your NLP pipeline.

Facebook Groups are actually more important than Twitter/X in rural West Africa. Mining-area community groups on Facebook (e.g. "Obuasi Residents Forum") are highly active and almost entirely ignored by existing monitoring tools because they require authenticated scraping. The CrowdTangle API (now Meta Content Library) gives academic/researcher access. This is a genuine data moat if you can get access.

Court and land registry records flag legal disputes over mining concessions early — often 12–18 months before they become headline news. These require a local legal data partner or manual extraction.

The combination of radio transcripts + Facebook group monitoring + parliamentary records gives you coverage that no international media monitoring tool currently provides for mining-affected communities in Africa. That's the real product.
